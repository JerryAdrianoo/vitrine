# Template — Desafio Curto

> Para exercícios pontuais (≤ 1h) que reforçam um conceito específico. Não confundir com projeto.

## 1. Conceito-alvo
- Qual conceito esse desafio reforça?
- Tier do roadmap

## 2. Enunciado
- Descrição clara do problema
- Restrições (entrada, saída, complexidade alvo)
- Exemplos de entrada → saída esperada

## 3. Critério de aceitação
- Compila e roda
- Passa nos exemplos
- Atende a complexidade alvo
- Lê bem (nomes, estrutura)

## 4. Pistas (em camadas)

### Pista 1 (mínima)
- Pergunta que ajuda a decompor o problema

### Pista 2 (média)
- Estrutura/algoritmo a considerar

### Pista 3 (forte)
- Esqueleto da abordagem (sem código pronto)

## 5. Casos de borda esperados
- Lista do que o aluno deve testar mentalmente

## 6. Solução de referência
- Apresentar **somente após** o aluno entregar a versão dele
- Comentar linha por linha com `//`
- Comparar com o que o aluno fez (sem juízo de valor)

## 7. Variação para fixar
- Mesmo padrão em contexto diferente — para não decorar
